## XPath-семантика контекста
Запрос выполняется в `ExecutionContext(documentRoot, contextNode)`:

- абсолютные пути (`/...`) считаются от `documentRoot` и в XPath-стиле ожидают,
  что первый шаг пути может совпадать с тегом корневого элемента документа
  (например документ `(root ...)` -> путь `/root/...`).

- относительные пути (`a/b`) считаются от `contextNode`.

`QueryEngine.find(root, path)` — сокращение, где `documentRoot=contextNode=root`.
В этом случае абсолютные пути относятся к этому же узлу (и начинаются с его тега).

### Примеры

#### Пример 1: абсолютный путь от корня документа (XPath-стиль)
Документ:
`(root (a) (b) (c))`

- Запрос `/` возвращает корневой узел документа:
    - найдено: `(root ...)`
    - находится по пути: `/root`

- Запрос `/root/a` находит узел `(a)`:
    - найдено: `(a)`
    - находится по пути: `/root/a`

- Запрос `/root/*` находит всех непосредственных детей `root`:
    - найдено: `(a)`, `(b)`, `(c)`
    - пути: `/root/a`, `/root/b`, `/root/c`

#### Пример 2: относительный путь от контекста
Документ:
`(root (maxverstappen (supermax :id 1) (supermax :id 33)) (tutududu))`

Контекст выполнения:
- `documentRoot` — весь документ `(root ...)`
- `contextNode` — узел `/root/maxverstappen` (то есть список `(maxverstappen ...)`)

Тогда:
- Относительный запрос `supermax` выполняется *от контекстного узла* и находит 2 узла:
    - найдено: два `(supermax ...)`
    - пути: `/root/maxverstappen/supermax` и `/root/maxverstappen/supermax`

- Абсолютный запрос `/root/tutududu` игнорирует контекст и ищет *от корня документа*:
    - найдено: `(tutududu)`
    - находится по пути: `/root/tutududu`
